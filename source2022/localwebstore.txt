    <h3> mX3 Engine framework POBST (Precompiled Object Based Scripting Tool)</h3>
      <h3>
          <strong>Extensible</strong></h3>
      <ul>
          <li><strong>OpenTools API</strong>&nbsp; precompiled functions of the App allows for pre and post processing of public methods and properties of the mX3 framework (anatomic) and mX4 (solar).&nbsp;
          E.G. you can alter title or form of the GUI or set your own menu, memo or edit style. You find the functions in /Debug/Code Completion List/ at the end of output. Check also the ex. 143_OpenTools_API.&nbsp; Full source code for the box and other Libs like SynEdit or ImageFileLibBCB is provided.</li>
          <li><strong>Provided Functions.</strong>&nbsp; Over 750 new Delphi, Pascal, network and Indy functions registered and built in (pre compiled) and functions/procedures and types/constants provided (see file: upsi_allfunctionslist.txt) or 
          The function Show Interface in context menu leverages this capability to implement a "Overview" command that returns a list on the current state of all used functions in a script.</li>
          <li><strong>Command Line Interface.&nbsp; </strong>The mX3 is fully command line enabled for both Win and Linux system, see example 150_command_line.</li>
      </ul>

    <h3>Procedure/Function Changes</h3>
    <p>
        Most apps or scripts will not be affected by this changes.&nbsp; However
        there are a some methods, properties, events, constants and enums that
        were removed or replaced with equivalent functionality from the VCL runtime lib, JEDI or Indy Globals.&nbsp; These changes were required for one of two reasons:</p>
      <ul>
          <li>Shift to using the more Pascal-like mX3 framework instead of the more c-like com interfaces that external libs used.</li>
          <li>Closer to the ObjectPascal Standard.&nbsp; Their is now a higher level of compatibility between native and CLX version of mX3.</li>
          <li>Most of the functions support now the Delphi Runtime Library (RTL) and you can call them directly from the script.</li>
          </ul>
      <p>
          <strong>Known compatibility issues and bugs</strong>:</p>
      <ul>
          <li>A couple of minor bug fixes and now includes some built in test cases. Now is now;) a TDateTime and not a string anymore. 16bit unit is deprecated, empty and consequently no longer needed. Solved Write bug for output&nbsp; Most
              of this units content has been moved to either the Sysutils or System units.</li>
          <li>Debuging and Decompile can be disturbed either by events or method handlers are in use (e.g. onClose, onDrawcell, onIdle..) or a property like form.style =[fsBold]. So <strong>uncomment</strong> this line to debug the code. LoadByte
  Currency Converter Str has a problem with the type currency
  <BR>        
New compiletime and component count support functions for controls, unused symbols or misformed files.<BR>SetFocus on a form wont't work in any case</li>
          <li>Voice function supported<strong>.&nbsp; Voice(), Say() or Speak(). </strong>These static com library linked into mX3 allows a script to speak.&nbsp; </li>
      </ul>
